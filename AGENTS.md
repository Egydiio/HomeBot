## HomeBot — agent guide

- **Stack & runtime**: Laravel app tailored for WhatsApp automation. Primary entrypoint is the POST `routes/api.php` webhook → `WebhookController` → `App\Services\Bot\BotRouter`. Queue/Redis/Postgres run via `docker-compose.yml` (app, nginx on 8000, postgres 5432, redis 6379, dedicated `queue` worker and `scheduler` containers).
- **Conversation state**: `App\Services\Bot\ConversationState` stores per-phone state/data in Redis with 30m TTL. State machine keys: `idle`, `waiting_image_type`, `waiting_classification`, `waiting_manual_value`, `waiting_confirmation`. Always clear state when flow ends; set both state and data when awaiting user input.
- **Routing logic**: `BotRouter::handle()` lowercases text, logs, handles global commands (`ajuda/help/oi/menu`, `saldo`), then branches by state. Idle behavior: if image with caption keywords (conta/luz/agua/etc) → `BillHandler`; image without clear caption → asks for type and stores media; otherwise delegates to `HelpHandler`.
- **Receipt flow**: `ReceiptHandler` creates `Transaction` type `receipt`, sets status `pending`, state `waiting_classification`, queues `ProcessReceiptImage`. Job downloads image, runs Google Vision → sends text to OpenAI (`gpt-4o-mini`) to extract JSON `{total, items[]}` with `category` house/personal rules, then populates `transaction->items` and calls `ClassifyHandler` to message user with house vs personal items and total. User reply `SIM` confirms; other values prompt correction.
- **Bill flow**: `BillHandler` creates `Transaction` type `bill`, sets state `waiting_manual_value`, and also dispatches `ProcessReceiptImage` on queue `ocr` to try to guess total. If OCR finds total, job asks for SIM/NÃO confirmation; else bot prompts for manual value (`handleManualValue`). Manual/confirmed values update `house_amount`, mark status `confirmed`, then call `BalanceService::updateBalance`.
- **Balance updates**: `BalanceService` runs inside DB transaction. For each active group member (except payer) it calculates share using `split_percent`, offsets inverse balances, and writes `balances` rows per reference_month (first day string). `getFormattedSummary` builds WhatsApp-friendly text; used by `BalanceHandler` when user sends `saldo`.
- **Month close & charging**: `homebot:close-month` command checks 5th business day via `BusinessDayService` (national holiday list) unless `--force`. For each group/member debt in the reference month, it creates `MonthlyClose` and dispatches `SendPixCharge`. Job tries MercadoPago `PixService::generatePaymentLink` (access token env), sends WhatsApp message with payment link + Pix fallback, marks close as `charged`; reminders via `SendPaymentReminder` reuse Pix fallback.
- **Outbound messaging**: `ZApiService` posts to Z-API using env `ZAPI_INSTANCE/TOKEN/CLIENT_TOKEN`. Prefer `sendText`; `sendImage` exists. All handlers rely on this; avoid bypassing.
- **Data model**: Key tables/models `Group`, `Member(split_percent, pix_key, active)`, `Transaction(items, type receipt|bill, status pending/processed/confirmed, reference_month)`, `TransactionItem(category house|personal, confirmed)`, `Balance(debtor_id, creditor_id, reference_month, amount)`, `MonthlyClose(status pending/charged/paid, debtor/creditor)`. Reference month stored as date (first day).
- **Queues & timeouts**: `ProcessReceiptImage` tries=3 timeout=120s; `SendPixCharge` and `SendPaymentReminder` tries=3 timeout=60s. Queue connection is Redis (`QUEUE_CONNECTION=redis`). Scheduler container runs `php artisan schedule:run` every minute; ensure new scheduled tasks are idempotent.
- **Config & secrets**: `.env` uses Postgres + Redis; Google Vision credentials path `storage/app/google-vision.json`; OpenAI key `OPENAI_API_KEY`; MercadoPago token `MERCADOPAGO_ACCESS_TOKEN`; Z-API tokens and instance ID; queue/redis hosts set to service names when running in compose.
- **Local workflows**: Typical bring-up: `docker-compose up -d app nginx postgres redis queue scheduler`; install deps in app container (`composer install`, `npm install && npm run build` if UI needed). Queue worker already defined in compose; if running locally without compose, start `php artisan queue:work` and `php artisan schedule:work`/cron. Inbound webhooks hit `http://localhost:8000/api/webhook` via Z-API.
- **Extending the bot**: Add new commands in `BotRouter::handle()` global commands block or new state branches. Reuse `ConversationState` to gate user input; always log and send friendly prompts via `ZApiService`. For new OCR-like flows, dispatch jobs and keep webhook handler fast.
